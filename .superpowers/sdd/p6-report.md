## Architecture

```
Request → NotificationService (gRPC/HTTP handler)
         → NotificationUsecase (biz logic)
              → PreferenceRepo (PostgreSQL via sqlc)
              → GotifyClient (HTTP POST to Gotify API)
```

### Domain layer (`biz`)

- `PreferenceRepo` — interface for CRUD on notification preferences
- `GotifySender` — interface for push notification delivery
- `NotificationUsecase` — orchestrates preference checks (push_enabled, category opt-in, quiet hours), template resolution, and Gotify sending

### Data layer (`data`)

- `PreferenceRepo` — sqlc-generated queries against `notification.preferences`
- `GotifyClient` — simple HTTP client calling `POST {url}/message?token={token}`

### Key behaviors

- **Preference gating**: skips push if `push_enabled=false` or category-specific toggle is off
- **Quiet hours**: suppresses all push notifications during user-defined time window
- **Template rendering**: uses Go `text/template` with param substitution (e.g. `{{.Streak}}`)
- **Priority mapping**: `streak_reminder=3`, `friend_activity=5`, `achievement=7`
